# Task Summary

Read `README.md` first. It is the main challenge brief.

Build the **IT Delivery Workspace**, a small software engineering workflow module for:

```txt
request / idea -> planning -> development -> QA/testing -> release -> follow-up
```

Core modules:

1. IT Work Items
2. QA Checks
3. Release Notes

Minimum target:

- authenticated work item CRUD
- PostgreSQL persistence
- useful frontend screens
- basic workflow states
- setup notes and demo

Strong target:

- backend workflow validation
- QA readiness rule
- release deployment rule
- score integration
- meaningful tests
- one creative feature that improves the engineering workflow

Required submission files:

- `AI_USAGE.md`
- `PROMPT_LOG.md`
- `DECISIONS.md`
